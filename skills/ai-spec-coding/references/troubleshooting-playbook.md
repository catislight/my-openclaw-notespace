# Troubleshooting Playbook

用于处理构建失败、CI 失败、依赖安装异常、环境差异、远端独有问题。

## 先判断是不是结构性问题

如果具备以下任一特征，按实验驱动排障，不要直接连续大改：
- 只能在 CI / 测试 / 生产复现
- 本地无法复现
- 日志不完整
- 一次验证很慢
- 怀疑多根因叠加
- 怀疑依赖 postinstall / native binding / 锁文件问题

## 标准排障流程

### 1. 固定问题表述

写清楚：
- 现象
- 首次出现时间
- 受影响环境
- 最近变更
- 已知不受影响环境

### 2. 收集环境差异

至少比对：
- Node 版本
- 包管理器版本
- lockfile
- 平台/架构
- 环境变量
- 安装命令
- 构建命令
- .npmrc / .yarnrc / pnpm 配置

### 3. 建立假设列表

格式：

```md
- 假设 A：
  - 依据：
  - 验证方式：
  - 预期结果：
```

### 4. 一次只验证一个变量

避免同时修改：
- 依赖版本
- 安装命令
- 构建命令
- 环境变量
- 锁文件

否则无法判断因果。

### 5. 必要时读依赖源码

当出现这些信号时，应考虑直接读依赖源码：
- 日志没有报错但卡死
- postinstall 行为异常
- 文档未解释实际行为
- 不同平台安装产物不一致

### 6. 输出结论时区分三层

- 直接现象
- 中间机制
- 最终根因
