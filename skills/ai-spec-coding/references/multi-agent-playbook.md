# Multi-Agent Playbook

当任务较大、可并行、且边界可拆分时使用。

## 角色划分

### Planner
负责：
- 澄清需求
- 输出 proposal / design / tasks
- 定义边界与验收标准

### Builder
负责：
- 按任务实现
- 保持与 spec 对齐
- 不擅自扩 scope

### Reviewer
负责：
- 检查规范一致性
- 检查边界条件
- 检查潜在技术债

### Verifier
负责：
- 执行测试、类型检查、冒烟
- 提供客观验证结果

### Documenter
负责：
- 输出变更说明
- 记录迁移与风险
- 整理验收结论

## 拆分原则

适合并行的任务：
- 独立模块
- 独立页面
- 独立 service/type 生成
- 实现与文档整理分离

不适合并行的任务：
- 同一核心文件的高耦合修改
- 依赖顺序极强的重构
- 尚未明确边界的需求

## 主控原则

必须有一个主控负责：
- 汇总状态
- 决定合并顺序
- 解决冲突
- 保持 spec 单一事实源

## 并行输出格式

每个 Agent 返回应至少包含：
- 已完成项
- 未完成项
- 阻塞项
- 需要主控决策项
