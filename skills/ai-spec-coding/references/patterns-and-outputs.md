# Patterns and Outputs

用于标准化、中颗粒需求。

## 1. 标准 CRUD 页面

推荐结构：

```text
page-name/
├── components/
├── hooks/
├── services/
├── constants/
├── types.ts
└── index.tsx
```

默认生成顺序：
1. 明确接口与字段
2. 生成 types
3. 生成 services
4. 生成页面 hooks
5. 生成页面与子组件
6. 绑定查询、分页、筛选、行操作
7. 加入空态、加载态、错误态
8. 补类型检查与冒烟验证

## 2. 标准表单页

至少明确：
- create / edit 是否共用
- 默认值来源
- 字段校验
- 提交前转换
- 提交后跳转/关闭/刷新

## 3. 标准列表页

至少明确：
- 查询条件
- 分页字段
- 排序能力
- 批量操作
- 行内操作
- 刷新策略

## 4. 组件生成默认要求

- Props 明确，不要透传一坨 any
- 状态最小化
- 事件命名统一
- 副作用收敛在 hooks 或容器层
- 展示组件尽量保持纯

## 5. Service 层默认要求

- 接口命名统一
- 请求/响应类型显式声明
- 错误处理策略一致
- 不在页面中直接拼接请求细节

## 6. 生成代码时的检查清单

生成后至少检查：
- 有没有 any / as unknown as 滥用
- 有没有把所有逻辑塞进单文件
- 有没有绕过既有封装直接另起炉灶
- 有没有漏空态/加载态/错误态
- 有没有命名和目录不一致
- 有没有旧 API / 废弃 API
